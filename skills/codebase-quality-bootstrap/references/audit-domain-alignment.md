# Audit Domain Alignment Matrix

This is the contract between `codebase-quality-bootstrap` and `codebase-audit`. Every CLAUDE.md rule generated by the bootstrap must trace to a specific audit domain and check it prevents. Analyzers use this matrix to ensure complete coverage.

## Coverage Summary

| Domain | Code | Analyzer | Rules Generated |
|--------|------|----------|----------------|
| Security | SEC | security-standards-analyzer | OWASP Top 10 prevention rules |
| Data Privacy | PRIV | security-standards-analyzer | PII handling, encryption, data minimization |
| Code Quality | QUAL | code-quality-analyzer | SOLID, DRY, naming, complexity, error handling |
| Dead Code | DEAD | code-quality-analyzer | Unused imports/functions/variables/files prohibition |
| Deprecated Patterns | DEPR | code-quality-analyzer | Current APIs only, no legacy patterns |
| Enterprise Mandates | MAND | code-quality-analyzer | 7 mandates verbatim |
| Architecture | ARCH | architecture-analyzer | Dependency direction, circular deps, coupling |
| Concurrency | CONC | architecture-analyzer | Shared state, locks, async patterns (conditional) |
| Test Coverage | TEST | testing-standards-analyzer | Coverage, assertions, edge cases, mock usage |
| Infrastructure | INFRA | infrastructure-analyzer | Containers, CI/CD, env vars, health checks |
| Dependencies | DEP | infrastructure-analyzer | Version pinning, CVEs, unused deps, lock files |
| Performance | PERF | infrastructure-analyzer | N+1 queries, bounded collections, resource leaks |
| Documentation | DOC | documentation-standards-analyzer | README, API docs, config docs, architecture docs |

## Detailed Rule-to-Check Mapping

### SEC (Security) -- Prevented by security-standards-analyzer

| CLAUDE.md Rule | Audit Check Prevented |
|---------------|----------------------|
| All database queries use parameterized statements | SEC: SQL injection (OWASP A05) |
| All user input validated at boundary before business logic | SEC: Input validation bypass (OWASP A05) |
| Secrets loaded from env vars or secret managers, never hardcoded | SEC: Hardcoded secrets (OWASP A04) |
| Use framework-specific security headers/middleware | SEC: Security misconfiguration (OWASP A02) |
| Passwords hashed with memory-hard algorithm (argon2, bcrypt, scrypt) | SEC: Weak cryptography (OWASP A04) |
| Authentication required for all non-public endpoints | SEC: Broken authentication (OWASP A07) |
| Authorization checked on every request, not just UI | SEC: Broken access control (OWASP A01) |
| CORS restricted to specific allowed origins | SEC: CORS misconfiguration (OWASP A02) |
| No sensitive data in error responses or stack traces | SEC: Information exposure (OWASP A10) |
| Dependencies scanned for known CVEs | SEC: Supply chain vulnerabilities (OWASP A03) |
| TLS required for all data in transit | SEC: Cleartext transmission (OWASP A04) |
| Session tokens use secure attributes (HttpOnly, Secure, SameSite) | SEC: Session hijacking (OWASP A07) |

| CLAUDE.md Rule | Audit Check Prevented |

### ARCH (Architecture) -- Prevented by architecture-analyzer

| CLAUDE.md Rule | Audit Check Prevented |
|---------------|----------------------|

| No circular dependencies between modules | ARCH: Circular dependency |
| Each module has a single, clear responsibility | ARCH: Low cohesion |
| Modules communicate through well-defined interfaces | ARCH: High coupling |

| CLAUDE.md Rule | Audit Check Prevented |
